# Test Plan

## 1. Introduction
This test plan describes the testing strategy for the selected web application.

## 2. Objectives
- Verify core functionalities work as expected
- Identify defects before release
- Ensure basic usability and stability

## 3. Scope

### In Scope
- Functional testing
- Manual testing
- Exploratory testing
- API testing (basic)

### Out of Scope
- Performance testing
- Security testing
- Automation (for now)

## 4. Test Environment
- Browser: Chrome, Firefox
- OS: Windows
- Tools: Jira, GitHub, Postman

## 5. Test Strategy
- Execute manual test cases
- Perform exploratory testing
- Report defects in Jira
- Validate APIs using Postman

## 6. Entry Criteria
- Application is accessible
- Test environment is stable

## 7. Exit Criteria
- All planned test cases executed
- Critical bugs reported
- Test results documented

## 8. Deliverables
- Test plan document
- Test cases
- Bug reports
- Test summary

## 9. Risks
- Limited test environment
- Time constraints
